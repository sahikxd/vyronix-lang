🚀 Quick Start

1. Install

Download binaries from GitHub Releases 
 and add them to PATH.

2. Create hello.vx

fn main() { 
     let name = "Developer"; 
     io.print("Hello, " + name + "!"); 
 } 
 main();

3. Run

vyronix run hello.vx

4. Compile Bytecode

vyronix build hello.vx -o hello.vyb 
 vyronixvm hello.vyb

5. Install Packages

vyx init 
 vyx install http

💻 Code Examples

1. Fibonacci Sequence

Recursive functions and basic math operations.

fn fib(n: i64) -> i64 { 
     if (n <= 1) return n; 
     return fib(n - 1) + fib(n - 2); 
 } 
 io.print(fib(10));

2. Closures & Higher-Order Functions

Functions returning functions while capturing local state.

fn multiplier(factor: i64) -> fn(i64) -> i64 { 
     return fn(n: i64) { 
         return n * factor; 
     }; 
 } 
 let double = multiplier(2); 
 io.print(double(5));

3. Structs

Custom data structures with fields.

struct Point { 
     x: f64, 
     y: f64 
 } 
 let p = Point { 
     x: 10.5, 
     y: 20.0 
 }; 
 io.print(p.x);

4. Error Handling

Simple try/catch support.

try { 
     throw "Something went wrong!"; 
 } catch (e) { 
     io.print("Caught error: " + e); 
 }

5. Generics

Reusable code for multiple data types.

fn identity<T>(value: T) -> T { 
     return value; 
 } 
 let a = identity(42); 
 let b = identity("Hello");

📖 Wiki & FAQ

Why C++17?

Because it gives raw performance with modern tooling. You get low-level control without writing prehistoric code like it's 2003.

Compiled or Interpreted?

Hybrid approach. VYRONIX compiles into .vyb bytecode, then executes through a custom stack VM.

How does vyx work?

The package manager uses GitHub repositories as a decentralized registry. Lightweight. Simple. No corporate monster infrastructure needed.

Can VYRONIX use native libraries?

Yes. Through experimental FFI support, you can call native DLLs and shared libraries.

What is Peephole Optimization?

The optimizer scans tiny instruction patterns and replaces wasteful operations with faster equivalents.
