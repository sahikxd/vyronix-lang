📖 Wiki & FAQ

Common questions and technical details about the VYRONIX ecosystem.

Q: Why C++17 for the implementation?
C++17 provides a perfect balance of modern features (like std::variant, std::optional, std::filesystem) and high performance, making it ideal for building a fast VM and compiler.
Q: Is VYRONIX compiled or interpreted?
VYRONIX uses a hybrid approach. It is compiled into custom bytecode (.vyb), which is then executed by a highly optimized stack-based Virtual Machine.
Q: How does the Package Manager work?
vyx is decentralized. It uses GitHub as a registry. When you run vyx install, it looks up the package in a static JSON file hosted on GitHub Pages and downloads the source directly from the repository.
Q: What makes the Optimizer "Peephole"?
A peephole optimizer looks at small windows of instructions (the "peephole") and replaces inefficient sequences with faster ones, such as removing redundant PUSH/POP pairs or folding constants.
Q: Can I use C++ libraries in VYRONIX?
Yes! Through our experimental FFI (Foreign Function Interface), you can load dynamic libraries (.dll/.so) and call native functions directly from VYRONIX code.
