# Contributing to VYRONIX

First off, thanks for taking the time to contribute! ❤️

## Code of Conduct
This project and everyone participating in it is governed by the [VYRONIX Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs
- Use the **Bug Report** template.
- Provide a clear description and steps to reproduce.

### Suggesting Enhancements
- Use the **Feature Request** template.
- Explain why this enhancement would be useful.

### Pull Requests
1. Fork the repo and create your branch from `dev`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code follows the existing style.
5. Write a clear PR description.

## Style Guide
- Use C++17 standards.
- Follow the naming conventions used in the codebase (PascalCase for classes, camelCase for functions/variables).
- Add comments for complex logic.

See the [README.md](README.md) for build instructions.
